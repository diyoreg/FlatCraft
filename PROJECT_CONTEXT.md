# FlatCraft v2 — автосборщик кухонь (3ds Max + веб-редактор)

## О проекте
Система автосборки кухонных интерьеров: 2D-редактор в браузере (перетаскивание
блоков) + сборщик 3D-сцены в 3ds Max 2024 (Corona Renderer).
Пользователь — дизайнер интерьеров, 5 лет опыта, фриланс.

## Направление v2 (ключевые решения)
- **Без техники**: 3D-модели (плита, холодильник и т.д.) пока не подгружаются —
  в раскладке резервируются проёмы (`gap`), технику пользователь вставляет вручную.
- **Простые блоки**: полные корпуса шкафов как параметрические боксы
  с нужными размерами и зазорами. Детализация — позже.
- **Веб-интерфейс**: минимализм, тёмная тема. 2D-редактор с видами со всех
  сторон (спереди / сверху / сбоку), drag&drop блоков, сохранение проекта.
- **Формат проекта**: JSON (см. `docs/project-format.md`) — общий язык
  веб-редактора и сборщика. Старые INI не мигрируются (чистый лист).
- **Локально**: всё работает на машине пользователя, без хостинга.
- **Max-сторона на Python** (pymxs): надёжный JSON, проще расширять.

## Структура репозитория
```
FlatCraft/
  builder/
    fc_builder.py       -- сборщик: project.json -> 3D-сцена (pymxs)
    fc_scene.py         -- сцена-на-проект: projects/<slug>/<slug>.max,
                           автосохранение при переключении, защита чужих сцен
    FC_Build.ms         -- запуск вручную из Max (диалог выбора JSON)
  launcher/
    setup.cmd           -- разовая настройка: находит Max (приоритет 2024),
                           пишет launch.cmd, регистрирует протокол flatcraft://
    FC_Startup.ms       -- стартовый скрипт Max: поднимает мост (если не занят
                           другим экземпляром) и выполняет отложенную сборку
    fc_startup.py       -- чтение bridge/launch_request.txt -> build_project
  bridge/
    FC_Bridge.ms        -- файловый мост: inbox/*.ms|*.py -> выполнение -> outbox/*.txt
  docs/
    project-format.md   -- спецификация формата project.json
  projects/
    sample/project.json -- тестовый проект (линейная кухня)
  legacy/
    FlatCraft_v1.ms     -- старый скрипт v1 (моно-файл с UI) — только справочно
  web/                  -- 2D-редактор (открывается файлом, без установки)
    index.html          -- разметка и стили (тёмный минимализм)
    core.js             -- чистая логика: раскладка, snap, undo, валидация
    app.js              -- UI: DnD, зум/пан, инспектор, БД проектов, мост
    selftest.js         -- автотесты: index.html?selftest=1 (title = TESTS OK)
```
Локально также лежат (в git не входят): `models/`, `textures/`,
`kitchen sample image hi tech/` — тяжёлые ассеты.

## Мост с 3ds Max (bridge/)
`FC_Bridge.ms` запускается в Max и раз в 400 мс выполняет файлы из
`bridge/inbox/` (*.ms через fileIn, *.py через python.ExecuteFile), результат
пишет в `bridge/outbox/<имя>.txt`, heartbeat — `bridge/alive.txt`.
Отправитель должен писать файл во временное имя и переименовывать в *.ms.
Остановка: `FC_BridgeStop()` в Listener. Позже этот же мост использует
веб-редактор для «живой» сборки.

## Технические требования
- 3ds Max 2024, системные единицы сцены = 1 мм (сборщик предупреждает, если нет).
- Corona: CoronaPhysicalMtl, фолбэк — PhysicalMaterial.
- Все объекты сборки: префикс `FC_`, слой `FlatCraft`; пересборка удаляет старые.
- .ms-файлы — только ASCII (кириллица в .py и .json, там UTF-8 без проблем).

2. [x] Формат проекта JSON + тестовый проект
3. [x] Файловый мост для прямого управления Max
4. [x] Проверка геометрии в Max (через мост: числа + скриншот вьюпорта)
5. [x] Веб-редактор v2 (`web/`): тёмный минимализм, виды спереди/сверху/сбоку,
       полный drag&drop (палитра -> чертёж, перестановка, ресайз с притяжкой
       к стандартным ширинам и краям соседнего ряда, удаление перетаскиванием),
       зум/панорама, undo/redo, контекстное меню, инлайн-ввод ширины,
       менеджер проектов (IndexedDB + фолбэк localStorage), автотесты
6. [x] Кнопка «Собрать в 3D» из браузера: File System Access API ->
       bridge/inbox -> fc_builder; мост защищён от недописанных файлов
       (файл выполняется при стабильном размере два тика подряд)
7. [x] Изоляция параллельных проектов: fc_scene.build_project(slug) держит
       каждую кухню в её собственной сцене projects/<slug>/<slug>.max;
       при переключении текущая сцена-проект автосохраняется (ручные правки
       и вставленная техника не теряются), посторонние несохранённые сцены
       не трогаются (сборка прерывается с ошибкой)
8. [x] Холодный старт из браузера: если мост молчит, кнопка пишет
       bridge/launch_request.txt и открывает flatcraft:// -> launch.cmd ->
       3ds Max 2024 c FC_Startup.ms (мост + отложенная сборка).
       Разовая настройка: launcher/setup.cmd. Индикатор «Max на связи» в шапке.
9. [x] Защита от случайной пересборки: после сборки отпечаток сцены
       (имя|класс|трансформация всех объектов) сохраняется в
       projects/<slug>/scene_state.json; при следующей сборке отличие ->
       ("confirm", детали) -> диалог в браузере -> повтор с force=True.
       Без ручных правок в Max пересборка идёт молча.
10. [x] Раскладки Г и П: kitchen.layoutType (linear|L|U) выбирается только
        при создании проекта; kitchen.sides = {south, west, east} — у каждой
        стороны свои offsetX/lower/upper. Сборщик отображает локальные
        координаты стороны в мировые (юг: как есть, запад: к x=0,
        восток: к x=length). Редактор: вкладки стен над видом спереди,
        вид сверху рисует все стороны (форма буквы), панель «Раскладка»
        правит активную стену, миниатюры проектов = план сверху.
        Шаблоны Г/П генерируются с логичной расстановкой (угол за главной
        стеной, крылья с отступом 644). Старый формат читается как linear.
7. [ ] Проёмы в стенах, другие стены, L/U-раскладки
8. [ ] Материалы/текстуры, стили фасадов, ручки; возврат техники (модели)
